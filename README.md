# shast LAB - GAS Web App (v4)

ラボ用バーコードスキャン入力スクリプト。フェーズ1(現場〜風乾)とフェーズ2(振り/ろか/分析)を一つのUIで操作。当日分の作業ログは日報シートに自動追記、ボタン1発で印刷可。

## 構成

| ファイル | 役割 |
|---|---|
| `Code.gs` | サーバー側ロジック (スキャン受付・各シートへの書込・上下混合判定・日報追記・PDF生成) |
| `Index.html` | ラボタブレットで開く入力UI |

## バーコード/モード対応表

| モード | スキャンするコード形式 | 書込先シート | 順番 |
|---|---|---|---|
| 採取 | `0001` (ベース) | 工程時刻 D列 + 担当者E | 厳格 |
| 受入 | `0001` | 工程時刻 F列 + 担当者G | 厳格(採取必須) |
| 風乾 | `0001` | 工程時刻 H列 + 担当者I | 厳格(受入必須) |
| 振り | `A1-1-R-huri` | 前処理 D列 + 担当者E | 柔軟(警告) |
| ろか | `A1-1-R-roka` | 前処理 F列 + 担当者G | 柔軟(振り) |
| 分析 | `A1-1-R-roka` (再利用) | **前処理 H列 + 担当者I** | 柔軟(ろか)、**早川 / 山口のみ** |

`-huri` / `-roka` 付きはモード未選択でも自動でモード切替されます。
**v4 から分析の書込先が「分析シート」→「前処理シート H/I 列」に変更されました。**

## 自動連携

### 上下混合 (工程時刻 → 前処理/分析 M列)
風乾モードで **同地点の上+下が両方完了** した瞬間に、混合地点名 (例: `A1-1`) を **前処理シート M列** と **分析シート M列** の末尾に自動追加します（重複チェック付き）。

その後の M→B 列展開、R/B/K の設定はスプレッドシート側の数式で行われます。

### バーコード作成シート (自動生成 / A4縦印刷向けレイアウト)
M列追加直後に `SpreadsheetApp.flush()` で B/C列の数式展開を確定させてから、
**前処理シート B/C列を読んで、その地点で C列が非空の色だけ** バーコード作成シートに追加します。

#### レイアウト (1地点 = 4行 × 6列)
横6カードを1地点ぶん横並びに配置。各カードは3行 + 地点間に空白行1行。

| 列 | A | B | C | D | E | F |
|---|---|---|---|---|---|---|
| QR行 | R-roka | B-roka | K-roka | R-huri | B-huri | K-huri |
| 色マーク行 | 赤 | 青 | 黒 | 赤 | 青 | 黒 |
| ラベル行 | A1-1 R ろか | A1-1 B ろか | A1-1 K ろか | A1-1 R 振り | A1-1 B 振り | A1-1 K 振り |
| 空白行 | (切り取り余白) | | | | | |

- 1行目はヘッダー (固定行)
- 8地点で約A4縦1ページに収まる
- C列にRしか無ければ R列だけ表示、他の色は空欄(歯抜けOK)
- QRは `=IMAGE("https://api.qrserver.com/v1/create-qr-code/?size=80x80&data=...", 4, 80, 80)` で実寸表示
  - 第2引数 `4` = 元サイズ厳守モード、第3/第4 = 高さ/幅(px)
  - (旧 chart.googleapis.com は廃止済みのため使わない)

#### 後からC列が編集された場合 → 手動再生成
GASエディタで関数 **`regenerateAllBarcodes`** を実行すると、
バーコード作成シートを一旦クリアし、前処理B列の全地点について再生成します。

### 日報シート (LABO1 リアルタイム記録)
スキャンのたびに当日付シート `yyyy-MM-dd` (例: `2026-05-07`) に1行追記されます。
- 対象工程: **受入 / 風乾 / 振り / ろか** の4工程
- 除外: 採取(現場側) / 分析(LABO2の別作業)
- シートが無ければ初回スキャン時に自動作成
- 列: A=地点 / B=上下またはRBK / C=工程 / D=日時 (yyyy/MM/dd HH:mm) / E=担当者

### 「本日分」印刷ボタン
担当者ボタン行の右端に大きめの🖨️ボタン。押すと:
1. 当日日報シートをPDFとして取得
2. 非表示iframeに流し込み
3. ブラウザの印刷ダイアログを起動

シートやPDFファイルは画面に出ず、shast LAB を離れずに完結。ファイルは保存されません。

## シート構造

### 工程時刻シート (本スクリプトが作成・書込)
| 列 | A | B | C | D | E | F | G | H | I |
|---|---|---|---|---|---|---|---|---|---|
| 内容 | 地点 | 上下 | コード | 採取 | 採取担当 | 受入 | 受入担当 | 風乾 | 風乾担当 |

A〜D列はバーコードシートからARRAYFORMULAで同期：
- A2: `=ARRAYFORMULA(IF(バーコード!A2:A="","",バーコード!A2:A))`
- B2: `=ARRAYFORMULA(IF(バーコード!B2:B="","",バーコード!B2:B))`
- C2: `=ARRAYFORMULA(IF(バーコード!C2:C="","",バーコード!C2:C))`
- D2: `=ARRAYFORMULA(IF(バーコード!D2:D="","",バーコード!D2:D))`

### 前処理シート (既存、別エンジニア管轄)
本スクリプトはM列(13列目)に地点名を追記するのみ。B列(地点名)、C列(R/B/K)展開は既存ロジックに依存。

書込列:
- 前処理: D=振り時刻 / E=振り担当 / F=ろか時刻 / G=ろか担当 / **H=分析時刻 / I=分析担当**

### 分析シート
v4では本スクリプトは書込みません(残置)。M列追加のみ既存通り行います。

### 日報シート (yyyy-MM-dd)
| 列 | A | B | C | D | E |
|---|---|---|---|---|---|
| 内容 | 地点 | 上下/RBK | 工程 | 日時 | 担当者 |

初回スキャン時に自動作成。ヘッダーは固定行1行。

## デプロイ

1. スプレッドシートを開く →「**拡張機能 → Apps Script**」
2. `Code.gs` の中身を本リポジトリの内容に貼り替え
3. ファイル「+」→ HTML → 名前 `Index` に本リポジトリの `Index.html` を貼り付け
4. 保存 → 「デプロイ → デプロイを管理 → ✏️編集 → バージョン: 新バージョン → デプロイ」
5. 表示されたWeb App URLをラボタブレットでブックマーク

初回デプロイ時、`UrlFetchApp` の権限承認(PDF取得用)が追加で求められます。

## 担当者管理

- 既定7名: 川村 / 山田 / 川添 / 石徹白 / 早川 / 田中 / **山口**
- ＋ボタンで動的に追加可能
- 選択中の担当者は ScriptProperties に永続化
- **分析モードは 早川 または 山口 のみ** 使用可

## 運用フロー

```
現場 (shast Android)
  ↓ 採取 → バーコードシート

ラボ受入時
  [採取/受入/風乾モード] でラボタブレットからスキャン
  ↓
  上+下 両方の風乾完了
  ↓
  shast LAB が前処理/分析シートM列に混合地点名を追加
  ↓
  既存システムが M→B 展開、R/B/K設定、QR生成

ラボ前処理時
  [振り/ろか モード] でA1-1-R-huri / A1-1-R-roka をスキャン → 前処理 D-G

ラボ分析時 (早川/山口のみ)
  [分析モード] でA1-1-R-roka をスキャン (ろかバーコード再利用) → 前処理 H/I

帰宅前
  [本日分] ボタン押下 → 当日日報シートを印刷
```

## トラブルシュート

### 「指定地点が見つかりません」
- 工程時刻シートのC列にベースコードがあるか確認
- 前処理シートのB列(地点名)+C列(色)が展開されているか確認
- 上下両方の風乾が完了していないとM列追加されない

### 「分析モードは 早川・山口 のみ使用できます」
- 担当者を 早川 または 山口 に切り替えてからスキャン

### 「本日の日報シートが見つかりません」
- まだ受入/風乾/振り/ろかを1件もスキャンしていない状態
- 1件スキャンすれば当日シートが自動作成される

### 印刷ダイアログが出ない
- ブラウザのポップアップブロックを確認
- フォールバックで PDF が新タブに開かれるので、Ctrl+P で印刷可能
