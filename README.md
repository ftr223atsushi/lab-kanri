# shast LAB - GAS Web App (v3)

ラボ用バーコードスキャン入力スクリプト。フェーズ1(現場〜風乾)とフェーズ2(振り/ろか/分析)を一つのUIで操作。

## 構成

| ファイル | 役割 |
|---|---|
| `Code.gs` | サーバー側ロジック (スキャン受付・各シートへの書込・上下混合判定) |
| `Index.html` | ラボタブレットで開く入力UI |

## バーコード/モード対応表

| モード | スキャンするコード形式 | 書込先シート | 順番 |
|---|---|---|---|
| 採取 | `0001` (ベース) | 工程時刻 D列 + 担当者E | 厳格 |
| 受入 | `0001` | 工程時刻 F列 + 担当者G | 厳格(採取必須) |
| 風乾 | `0001` | 工程時刻 H列 + 担当者I | 厳格(受入必須) |
| 振り | `A1-1-R-huri` | 前処理 D列 + 担当者E | 柔軟(警告) |
| ろか | `A1-1-R-roka` | 前処理 F列 + 担当者G | 柔軟(振り) |
| 分析 | `A1-1-R-roka` (再利用) | 分析 D列 + 担当者E | **早川のみ**、ろか未完了で警告 |

`-huri` / `-roka` 付きはモード未選択でも自動でモード切替されます。

## 自動連携

風乾モードで **同地点の上+下が両方完了** した瞬間に、混合地点名 (例: `A1-1`) を **前処理シート M列** と **分析シート M列** の末尾に自動追加します（重複チェック付き）。

その後の M→B 列展開、R/B/K の設定、QRラベル作成は別スクリプト管轄です。

## シート構造

### 工程時刻シート (本スクリプトが作成・書込)
| 列 | A | B | C | D | E | F | G | H | I |
|---|---|---|---|---|---|---|---|---|---|
| 内容 | 地点 | 上下 | コード | 採取 | 採取担当 | 受入 | 受入担当 | 風乾 | 風乾担当 |

A〜D列はバーコードシートからARRAYFORMULAで同期：
- A2: `=ARRAYFORMULA(IF(バーコード!A2:A="","",バーコード!A2:A))`
- B2: `=ARRAYFORMULA(IF(バーコード!B2:B="","",バーコード!B2:B))`
- C2: `=ARRAYFORMULA(IF(バーコード!C2:C="","",バーコード!C2:C))`
- D2: `=ARRAYFORMULA(IF(バーコード!D2:D="","",バーコード!D2:D))`

### 前処理シート / 分析シート (既存、別エンジニア管轄)
本スクリプトはM列(13列目)に地点名を追記するのみ。B列(地点名)、C列(R/B/K)展開は既存ロジックに依存。

書込列:
- 前処理: D=振り時刻 / E=振り担当 / F=ろか時刻 / G=ろか担当
- 分析: D=分析時刻 / E=分析担当

## デプロイ

1. スプレッドシートを開く →「**拡張機能 → Apps Script**」
2. `Code.gs` の中身を本リポジトリの内容に貼り替え
3. ファイル「+」→ HTML → 名前 `Index` に本リポジトリの `Index.html` を貼り付け
4. 保存 → 「デプロイ → デプロイを管理 → ✏️編集 → バージョン: 新バージョン → デプロイ」
5. 表示されたWeb App URLをラボタブレットでブックマーク

## 担当者管理

- 既定6名: 川村 / 山田 / 川添 / 石徹白 / 早川 / 田中
- ＋ボタンで動的に追加可能
- 選択中の担当者は ScriptProperties に永続化
- **分析モードは 早川 のみ** 使用可

## 運用フロー

```
現場 (shast Android)
  ↓ 採取 → バーコードシート

ラボ受入時
  [採取/受入/風乾モード] でラボタブレットからスキャン
  ↓
  上+下 両方の風乾完了
  ↓
  shast LAB が前処理/分析シートM列に混合地点名を追加
  ↓
  既存システムが M→B 展開、R/B/K設定、QR生成

ラボ前処理時
  [振り/ろか モード] でA1-1-R-huri / A1-1-R-roka をスキャン

ラボ分析時 (早川のみ)
  [分析モード] でA1-1-R-roka をスキャン (ろかバーコード再利用)
```

## トラブルシュート

### 「指定地点が見つかりません」
- 工程時刻シートのC列にベースコードがあるか確認
- 前処理/分析シートのB列(地点名)+C列(色)が展開されているか確認
- 上下両方の風乾が完了していないとM列追加されない

### 「分析モードは早川さんのみです」
- 担当者を早川に切り替えてからスキャン
